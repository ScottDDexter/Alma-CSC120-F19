Loop Patterns

The book talks about a few useful loop patterns, though it doesn't call them patterns, exactly (sometimes you'll see this kind of thing referred to as a loop idiom):

  • Input validation. Here, we read an input value, then enter a loop that repeats until the user actually gives us a valid value.
  • Sentinels. Here, the loop repeats until a particular value is encountered.
  • Accumulators (which is a general form of what the book calls "running sum"). Here, the loop progresses through a series of values and "accumulates" them somehow, most commonly by adding them.
  • Counting loops. Here, the loop keeps track of how many times it has repeated. This could either be a "definite" counting loop, where we count the repetitions so that we can stop after a fixed number (this is what a for loop is often used for), or an "indefinite" counting loop, where the stopping point isn't known ahead of time, and the whole point of the loop is to count how repetitions were required (for example, how many times can we cut n in half before the result is 1?)

There are many examples of these in chapter 11 of The Java Files; consider this an assigned reading for the weekend! We'll discuss several of these on Monday.

Two Programs

For the rest of today's lab, work on two programs. First, do PP 4.8 from the book—implementing a simple guessing game with a sentinel value indicating the user wants to stop guessing.

Then, you'll be able to use a lot of that basic program structure to tackle exercise 6 on this page. We'll adapt the exercise that we read data from the user (the keyboard), and we'll also add input validation that prints a message about any badly structured line of input, ignores the line, and continues prompting the user for input. I'll point out a few things, but as you get started, here's how I recommend you approach this:

  • First, write a pretty simple version of this in which each line of input is just a name or "stop." Print the names but not the sentinel value.
  • Once you have that working, add the ability to read name-plus-height from each line. For now, assume all the lines are correctly formatted. Print each nane and height but not the sentinel.
  • Once you have the input working, add the average calculations (average boy height, average girl height, average of all). (Feel free to replace "boy" and "girl" with less reductive categories—maybe these are dogs and cats, or robots and aliens.)
  • Now add logic that detects "bad" input and ignores it.
