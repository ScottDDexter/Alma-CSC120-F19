Objectives:

  • Use jGRASP to "trace" the progress of a program with conditional statements, and to evaluate condition expressions "on the fly."
  • Practice choosing what kind of conditional to use for a given problem.
  • Practice writing programs that use conditional statements.

Tackling Conditionals

Before we start working on problems, let's talk a little about how to approach this kind of problem. Deciding what kind of conditional statement to use—and then figuring out how to implement it—requires having a pretty solid understanding of the decision (or decisions) your program is going to have to make. It can be really useful to make diagrams of these choices before you start coding. These diagrams can also help clarify things like what variables you might need.

We'll work a few of these together, and along the way I'll show one important variant of nested if-else statements that the book doesn't address.

Small Problems

Positive and Negative

Write a Java program that reads an integer value from the user and prints whether it is positive or negative. ("Your number is positive." or "Your number is negative.").

Weekdays

Write a Java program that reads a number from the user, between 1 and 7, and prints out the name of the corresponding weekday (start with Sunday corresponding to 1).

Also, write a program that does the opposite: it reads a string from the user and prints out "That is the _____ day of the month" where the blank is "first", "second", and so on. Once you get that working, add behavior so that if the user's input is not the name of a weekday, your program prints a helpful message like, "That's not the name of a weekday."

Floating-point Equality

Write a Java program that reads in two floating-point numbers and tests whether they are the same up to three decimal places. So, 12.3324 and 12.3327 are "the same" but 12.3331 is different from both.

Floating-point Sizes

Write a Java program that reads a floating-point number and prints "zero" if the number is zero. Otherwise, print "positive" or "negative". Add "small" if the absolute value of the number is less than 1, or "large" if it exceeds 1,000,000. So, if the input value is -0.43, your program should output "negative small".

Leap year

Write a Java program that reads a year from the user and decides whether it is a leap year. Years before 1582 are not leap years. Otherwise, years that are divisible by 4 are leap years. Except that years divisible by 100 are not leap years. Except that years divisible by 400 are leap years. (This one benefits from thinking carefully about how to structure your conditional statement.)

Days in a Month

Write Java program that reads in a month number (1 to 12, corresponding to January to December), and a year, and prints out the number of days in that month. Remember: 

Thirty days have November,
April, June, and September.
With 28 there is but one.
All the rest have thirty-one.
And of course sometimes that 28 is 29....

Bonus: Quadratic Equation/Formula

You probably remember quadratic equations from math. And maybe even the quadratic formula. Write a program that reads three coefficients (a, b, and c) from the user and uses the quadratic formula to calculate the solutions of the quadratic equation ax2 + bx + c. The tricky part is the square root--what if the value under the square root is zero or negative?
